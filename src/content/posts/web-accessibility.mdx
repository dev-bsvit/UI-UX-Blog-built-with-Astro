# Веб-доступність: створюємо інклюзивні інтерфейси

Веб-доступність (Web Accessibility) — це практика створення веб-сайтів та додатків, які можуть використовувати люди з різними можливостями та обмеженнями. Це не просто вимога законодавства, а моральний обов'язок кожного дизайнера та розробника.

## Чому важлива доступність?

### Статистика
- **15%** світового населення мають інвалідність
- **285 мільйонів** людей мають проблеми зі зором
- **466 мільйонів** людей мають проблеми зі слухом
- **Більше 1 мільярда** людей мають різні обмеження

### Бізнес-переваги
- **Збільшення аудиторії** — більше потенційних користувачів
- **Покращення SEO** — доступні сайти краще індексуються
- **Відповідність закону** — уникнення правових проблем
- **Позитивний брендинг** — демонстрація соціальної відповідальності

## Основні принципи WCAG 2.1

WCAG (Web Content Accessibility Guidelines) — це міжнародний стандарт доступності.

### 1. Відчутний (Perceivable)
Інформація повинна бути представлена способами, які користувачі можуть сприйняти.

#### Ключові вимоги:
- **Альтернативний текст** для зображень
- **Підписи** для відео та аудіо
- **Адаптивний контент** — можливість зміни розміру тексту
- **Контрастність** — мінімум 4.5:1 для звичайного тексту

### 2. Функціональний (Operable)
Користувачі повинні мати можливість взаємодіяти з інтерфейсом.

#### Ключові вимоги:
- **Клавіатурна навігація** — повна функціональність без миші
- **Достатній час** — можливість налаштування таймерів
- **Уникнення спазму** — не використовувати елементи, що спазмують
- **Навігація** — зрозуміла структура та орієнтири

### 3. Зрозумілий (Understandable)
Інформація та функціональність повинні бути зрозумілими.

#### Ключові вимоги:
- **Читабельність** — зрозуміла мова та структура
- **Передбачуваність** — консистентна поведінка елементів
- **Допомога при помилках** — зрозумілі повідомлення про помилки
- **Ідентифікація** — чітке позначення елементів

### 4. Міцний (Robust)
Контент повинен бути сумісним з різними технологіями.

#### Ключові вимоги:
- **Семантична розмітка** — правильне використання HTML тегів
- **ARIA атрибути** — додаткова інформація для скрін-рідерів
- **Тестування** — перевірка з різними асистивними технологіями

## Практичні поради для дизайнерів

### Кольорова доступність

#### Контрастність
```css
/* Хороший контраст */
.text-primary {
  color: #1f2937; /* Темно-сірий */
  background: #ffffff; /* Білий */
  /* Контраст: 15.6:1 */
}

/* Поганий контраст */
.text-secondary {
  color: #6b7280; /* Світло-сірий */
  background: #ffffff; /* Білий */
  /* Контраст: 3.2:1 - недостатньо! */
}
```

#### Не покладайтеся тільки на колір
```html
<!-- Погано -->
<div class="error">Помилка</div>

<!-- Добре -->
<div class="error">
  <span aria-label="Помилка">⚠️</span>
  Помилка
</div>
```

### Типографіка

#### Розміри тексту
```css
/* Мінімальний розмір */
body {
  font-size: 16px; /* Базовий розмір */
  line-height: 1.5; /* Відступ між рядками */
}

/* Можливість збільшення */
.text-large {
  font-size: 1.25rem; /* Відносний розмір */
}
```

#### Вибір шрифтів
- **Sans-serif** шрифти краще читаються
- **Мінімум 16px** для основного тексту
- **Line-height 1.4-1.6** для комфортного читання

### Інтерактивні елементи

#### Розміри touch-targets
```css
/* Мінімальний розмір для мобільних */
.button {
  min-width: 44px;
  min-height: 44px;
  padding: 12px 16px;
}
```

#### Фокус та стани
```css
/* Видимий фокус */
.button:focus {
  outline: 2px solid #3b82f6;
  outline-offset: 2px;
}

/* Hover стани */
.button:hover {
  background-color: #1e40af;
}
```

## Інструменти для тестування доступності

### Автоматичні інструменти
- **axe DevTools** — браузерне розширення
- **WAVE** — онлайн аналіз
- **Lighthouse** — вбудований в Chrome DevTools
- **pa11y** — командний рядок

### Ручне тестування
- **Клавіатурна навігація** — Tab, Shift+Tab, Enter, Space
- **Скрін-рідери** — NVDA (Windows), VoiceOver (Mac)
- **Зум** — перевірка при 200% збільшенні
- **Кольорова сліпота** — симуляція різних типів

### Приклад тестування з клавіатури
```javascript
// Перевірка навігації
document.addEventListener('keydown', (e) => {
  if (e.key === 'Tab') {
    console.log('Tab navigation works');
  }
});
```

## ARIA (Accessible Rich Internet Applications)

### Основні ARIA атрибути
```html
<!-- Роль елемента -->
<button role="tab" aria-selected="true">Вкладка 1</button>

<!-- Опис елемента -->
<input aria-describedby="password-help" type="password" />
<div id="password-help">Мінімум 8 символів</div>

<!-- Стан елемента -->
<div aria-expanded="false" aria-controls="menu">
  Меню
</div>
```

### Поширені патерни
```html
<!-- Навігаційне меню -->
<nav aria-label="Головна навігація">
  <ul role="menubar">
    <li role="none">
      <a role="menuitem" href="/">Головна</a>
    </li>
  </ul>
</nav>

<!-- Форма -->
<form>
  <label for="email">Email:</label>
  <input 
    id="email" 
    type="email" 
    aria-required="true"
    aria-invalid="false"
  />
</form>
```

## Тестування з користувачами

### Планування тестування
1. **Рекрутинг** — знайдіть користувачів з різними потребами
2. **Підготовка** — створіть сценарії тестування
3. **Проведення** — спостерігайте та документуйте
4. **Аналіз** — виявіть проблеми та рішення

### Типові завдання
- Навігація по сайту
- Заповнення форм
- Пошук інформації
- Взаємодія з інтерактивними елементами

## Висновки

Веб-доступність — це не додаткова функція, а фундаментальна частина якісного дизайну. Створюючи доступні інтерфейси, ми:

- **Покращуємо досвід** для всіх користувачів
- **Розширюємо аудиторію** продукту
- **Демонструємо професіоналізм** та етичність
- **Відповідаємо стандартам** та законодавству

**Пам'ятайте**: доступний дизайн — це хороший дизайн для всіх.
